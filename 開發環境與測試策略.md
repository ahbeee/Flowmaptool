# Flowmaptool 開發環境與測試策略

文件版本：v0.2  
日期：2026-04-29

---

## 1. 目標

Flowmaptool 是 Windows local-first 桌面心智圖 / flow map 編輯器。開發策略是讓大部分品質檢查可以由本機命令完成，盡量降低手動驗收負擔。

---

## 2. 開發環境版本

## 2.1 作業系統

- Windows 11 x64

## 2.2 核心工具

- Node.js `24.15.0`
- pnpm `9.12.3`
- Git `2.47+`

## 2.3 建置相依

- Visual Studio 2022 Build Tools（Desktop development with C++）
- Python `3.12.x` 或目前可用的本機 Python

## 2.4 建議 IDE

- VS Code 最新版
- Playwright extension 可選，用來檢視 trace

---

- Electron `31.7.7`
- React `18.3.1`
- TypeScript `5.6.3`
- Vite `5.4.11`

## 4. 測試策略

## 4.1 測試分層

- Unit：graph、history、layout、local reflow、subflow 純邏輯測試。
- Integration：`.qflow` serialization 與舊版本 migration。
- E2E：Electron UI 行為、鍵盤滑鼠互動、工具欄、檔案操作、輸出、效能與錯誤提示。

## 4.2 本機測試命令

```powershell
pnpm lint
pnpm test:unit
pnpm test:integration
pnpm test:e2e
pnpm test:all
```

`pnpm test:e2e` 會先執行 production build，再用 Playwright 啟動 Electron 驗證。

## 4.3 目前 E2E 覆蓋重點

- App 啟動 smoke test。
- Tab / Enter / Space / Delete / Ctrl+C / Ctrl+V / arrow key 等快捷鍵。
- Node 編輯、選取切換、框選、多選刪除、root 拖曳、reparent。
- Horizontal / vertical layout、間距設定、root merge、多父節點、固定間距 reflow。
- 手動畫線、edge 選取、edge bend、route point、reset bend。
- 右側 toolbar 顯示/隱藏、map style、node style、tag、edge style。
- Zoom 後框選座標校正、fit to graph。
- PNG export quality。
- 壞檔案、格式錯誤、舊版檔案 migration UX。
- 500 nodes / 1000 edges 大圖效能壓測。

## 4.4 失敗時輸出

Playwright 失敗時會保留：

- screenshot
- trace zip
- 失敗測試的 test-results 目錄

## 5. Windows 打包

```powershell
pnpm pack:win
pnpm dist:win
```

- `pnpm pack:win`：產生 unpacked Windows app，適合快速本機確認。
- `pnpm dist:win`：產生 x64 portable 與 NSIS installer，輸出到 `release/`。
- `release/` 不進 git。
- 目前 build 未簽章。

---

## 6. 遠端 CI 狀態

依目前需求，不使用 GitHub Actions CI。測試與打包以本機命令為主：

```powershell
pnpm test:all
pnpm dist:win
```
