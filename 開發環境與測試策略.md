# flowmaptool 開發環境與測試策略（先準備清單）

文件版本：v0.1  
日期：2026-04-23

---

## 1. 目標

你先把環境準備好，之後我可以直接在本機執行大部分測試，盡量不需要你手動操作。

---

## 2. 開發環境版本（建議固定）

## 2.1 作業系統
- Windows 11 x64

## 2.2 核心工具
- Node.js `24.15.0` LTS（依 Node.js Releases 頁面「Latest LTS」）
- 套件管理：`pnpm 9.12.x`
- Git `2.47+`

## 2.3 建置相依（避免 native module 卡住）
- Visual Studio 2022 Build Tools（Desktop development with C++）
- Python `3.12.x`（給 node-gyp）

## 2.4 建議 IDE
- VS Code 最新版（含 ESLint、Prettier、Playwright extensions）

---

## 3. 專案技術棧（規劃）

- Electron `31.x`
- React `18.3.x`
- TypeScript `5.6.x`
- Vite `5.x`（renderer）
- Zustand（UI state）+ 自有 graph model store
- ELK.js（自動排版；horizontal/vertical）

---

## 4. 測試策略（你盡量不用參與）

## 4.1 測試分層
- Unit：model/operation 純函式測試（Vitest）
- Integration：serialization + layout fixtures（Vitest）
- E2E：Electron UI 行為（Playwright）
- Export smoke：PNG/PDF 匯出可開啟與尺寸檢查

## 4.2 我可以直接執行的腳本目標
- `pnpm lint`
- `pnpm test:unit`
- `pnpm test:integration`
- `pnpm test:e2e`（headless）
- `pnpm test:all`（單指令跑完）
- `pnpm build:win`

## 4.3 失敗時輸出
- 自動保存：
  - Playwright screenshot
  - Playwright trace zip
  - console/network logs
- CI artifact 保留最近 20 次

---

## 5. CI 規劃（Windows-only）

使用 GitHub Actions（`windows-latest`）：

1. 安裝 Node / pnpm  
2. 安裝相依  
3. `lint + unit + integration`  
4. `e2e (headless)`  
5. `build:win`  
6. 上傳 artifacts（安裝包、測試報告、trace）

---

## 6. 你現在要先做的事（一次到位）

1. 安裝 Node.js `24.15.0` LTS  
2. 安裝 pnpm：`npm i -g pnpm@9`  
3. 安裝 VS 2022 Build Tools（C++ workload）  
4. 安裝 Python `3.12.x`  
5. 確認：
   - `node -v`
   - `npm -v`
   - `pnpm -v`
   - `python --version`

---

## 7. 版本選擇說明（簡短）

- 不選 Node 20：已接近/進入 EOL 區間，後續相依風險高。  
- 選 Node 24 LTS：長期維護週期較好，較適合作為新專案基線。  
- 先固定版本：可降低「你本機能跑、我這邊不能跑」這類環境偏差。
